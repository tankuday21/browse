# V3 Phase 3 — Reading Stack — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Andromeda into a first-class reading browser — spec items H1 H3 H5 H6 H7: offline reading list, read-aloud with podcast mode, print/save-as-PDF, per-site display memory, and a polished reader with font/theme/width controls.

**Architecture:** Reader extraction stays a JS heuristic in `ReaderMode` (H7 improves it: link-density rejection, title dedupe, nav/aside pruning). Saved articles store the *unthemed* extracted content HTML on disk (`filesDir/reading_list/<id>.html`) via a small `ArticleStore`; theming happens at render time so reader controls apply to saved articles too. Room migration **v7→v8** adds `reading_list` + `site_settings` in ONE migration (per spec §4). Read-aloud is a foreground `ReadAloudService` (platform `TextToSpeech` + `MediaSession`, no new runtime deps) fed by pure `TtsChunker` (sentence chunks under the TTS input limit) and `TtsQueue` (podcast ordering over unread items). Per-site memory is a pure `SiteSettingsResolver` (global setting + tri-state override → effective value) applied in `WebViewHolder` at page start, edited from a "Site settings" sheet in the menu. Print uses the framework `PrintManager` + `WebView.createPrintDocumentAdapter` (system dialog includes Save-as-PDF).

**Tech Stack:** Kotlin, coroutines, Room+KSP, Compose M3, `android.speech.tts.TextToSpeech`, `android.media.session.MediaSession`. **No new Gradle dependencies.**

**Interfaces produced (verbatim):**

```kotlin
@Entity(tableName = "reading_list")
data class ReadingListEntry(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val url: String,
    val title: String,
    val addedAt: Long,
    val readAt: Long? = null,     // null = unread
    val filePath: String? = null, // offline cleaned-content HTML; null = online-only
)

@Dao interface ReadingListDao {
    @Query("SELECT * FROM reading_list ORDER BY addedAt DESC") fun observeAll(): Flow<List<ReadingListEntry>>
    @Query("SELECT * FROM reading_list WHERE id = :id") suspend fun getById(id: Long): ReadingListEntry?
    @Query("SELECT * FROM reading_list WHERE readAt IS NULL ORDER BY addedAt ASC") suspend fun getUnread(): List<ReadingListEntry>
    @Insert suspend fun insert(entry: ReadingListEntry): Long
    @Query("UPDATE reading_list SET readAt = :readAt WHERE id = :id") suspend fun setReadAt(id: Long, readAt: Long?)
    @Query("UPDATE reading_list SET filePath = :filePath WHERE id = :id") suspend fun setFilePath(id: Long, filePath: String?)
    @Query("DELETE FROM reading_list WHERE id = :id") suspend fun deleteById(id: Long)
    @Query("SELECT EXISTS(SELECT 1 FROM reading_list WHERE url = :url)") suspend fun existsByUrl(url: String): Boolean
}

@Entity(tableName = "site_settings")
data class SiteSettingsEntity(
    @PrimaryKey val host: String,
    val textZoom: Int = -1,     // -1 unset, else 50..200 (percent)
    val forceDark: Int = -1,    // -1 default, 0 off, 1 on
    val desktopMode: Int = -1,  // -1 default, 0 off, 1 on
)

@Dao interface SiteSettingsDao {
    @Query("SELECT * FROM site_settings WHERE host = :host") suspend fun getByHost(host: String): SiteSettingsEntity?
    @Query("SELECT * FROM site_settings") fun observeAll(): Flow<List<SiteSettingsEntity>>
    @Insert(onConflict = OnConflictStrategy.REPLACE) suspend fun upsert(entity: SiteSettingsEntity)
    @Query("DELETE FROM site_settings WHERE host = :host") suspend fun deleteByHost(host: String)
}
```

`MIGRATION_7_8` creates both tables with SQL matching Room's expected schema exactly (autoincrement PK for reading_list, TEXT PK NOT NULL for site_settings, NOT NULL + defaults per the data classes).

```kotlin
object HtmlText { fun strip(html: String): String }  // tags→text, block tags→sentence breaks, entities decoded

object TtsChunker { fun chunk(text: String, maxLen: Int): List<String> }

class TtsQueue(items: List<Item>) {           // Item(id: Long, title: String)
    data class Item(val id: Long, val title: String)

```

**Behavior:** `ReadAloudService` — START intent carries either (`articleId` for a saved item) or (`title` + content handed via a static holder for the live reader page, same pattern as MediaHoldService's controller). Foreground notification (mediaPlayback) with MediaSession transport: play/pause, next-article (podcast mode), stop; speed cycle action 1.0→1.25→1.5→0.75 (`setSpeechRate`, restart current chunk). Podcast mode: queue = unread reading list (oldest first); each finished article gets `readAt=now`; advances until queue dry, then stops itself. Utterance progress via `UtteranceProgressListener`; screen-off playback is inherent (foreground service). Static controller/callbacks nulled on destroy (MediaHoldService leak lesson). TTS init failure → toast + stopSelf, never a crash.
