# v6.10 — Bookmark folders

**Goal:** Let the user organize bookmarks into named folders and see them grouped on the Bookmarks
screen. The `bookmarks.folder` column has existed since Phase 2c but is never written or shown —
this surfaces it.

**Why:** A flat, growing bookmark list is hard to navigate. Folders are table-stakes for a
bookmarks manager. The schema and a `setFolder` DAO query already exist; only the VM wiring and UI
are missing.

## Design

No schema change (the `folder: String?` column already exists; `BookmarkDao.setFolder` already
exists). Everything is additive.

### `browser/BookmarkFolders.kt` (pure, JVM-tested)
```
object BookmarkFolders {
    /** Trim; blank -> null (top level). Collapses whitespace-only names to "no folder". */
    fun normalize(name: String?): String?

    /** Distinct folder names present, case-insensitively sorted (for the picker + section order). */
    fun folders(bookmarks: List<Bookmark>): List<String>

    /**
     * Ordered sections for display: each named folder (alpha, case-insensitive) with its bookmarks,
     * then a trailing null-key section for top-level (folderless) bookmarks. Empty sections omitted;
     * bookmarks within a section keep their incoming order (newest-first from the DAO).
     */
    fun sections(bookmarks: List<Bookmark>): List<Pair<String?, List<Bookmark>>>
}
```

### `BrowserViewModel.kt`
```
/** Assign (or clear, folder = null) the folder of the active Orbit's bookmark for [url]. */
fun onSetBookmarkFolder(url: String, folder: String?) {
    val orbitId = activeOrbitId.value
    val norm = BookmarkFolders.normalize(folder)
    viewModelScope.launch { bookmarkDao.setFolder(orbitId, url, norm) }
}
```

### `ui/BookmarksScreen.kt`
- Group the list via `BookmarkFolders.sections(bookmarks)`: render a folder header row per named
  folder (Folder icon + name + count), then its bookmark rows; the top-level section renders with no
  header (or a subtle "Ungrouped" label only when named folders also exist).
- Folders are **collapsible** — a header taps to expand/collapse; collapse state is local screen
  state (`remember`), keyed by folder name, defaulting to expanded.
- Each `BookmarkRow` gains a "Move to folder" affordance (an overflow/folder icon) opening a
  dropdown: **None** (clears), each existing folder, and **New folder…** → a small text dialog
  (`OrbitTextField`) that calls `onSetBookmarkFolder(url, entered)`. Reuses the existing delete
  action.
- Empty state unchanged.

### Incognito / Orbits
Bookmarks are already Orbit-scoped and never created in incognito; folders inherit that (the
`setFolder` query is keyed by `(orbitId, url)`). No new isolation surface.

## Testing
`BookmarkFoldersTest` (pure):
- `normalize`: trims, blank/whitespace → null, keeps a real name.
- `folders`: distinct, case-insensitive sort, ignores null-folder bookmarks.
- `sections`: named folders alpha-first each with their rows, top-level section last, empty
  omitted, in-section order preserved; all-top-level → single null section; all-foldered → no null
  section.
`BrowserViewModelTest`:
- `onSetBookmarkFolder` writes the normalized folder for the right (orbit, url); blank → null.

## Non-goals
Nested/sub-folders; drag-and-drop between folders; folder rename/delete as first-class objects (a
folder exists exactly while ≥1 bookmark references it — clearing the last bookmark's folder makes it
disappear, which is the same model Chrome-lite bookmark bars use); per-folder ordering.
