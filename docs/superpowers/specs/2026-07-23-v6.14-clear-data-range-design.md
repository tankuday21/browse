# v6.14 — Clear browsing data by time range

**Goal:** Let the user clear history for a chosen time range — Last hour / Last 24 hours / All time
— instead of the current all-or-nothing wipe.

**Why:** "Clear the last hour" is a common, low-stakes privacy action (you just visited something you
don't want kept). Chrome/Firefox both offer a time-range selector. Today Andromeda's "Clear browsing
data" only clears everything.

## Design

Add a range to the existing Settings → "Clear browsing data" dialog. History deletion becomes
range-scoped; the WebView HTTP cache has **no time-range API**, so it is always fully cleared
regardless of range (stated in the dialog copy).

### `browser/ClearDataRange.kt` (pure, JVM-tested)
```
enum class ClearDataRange(val label: String) {
    LAST_HOUR("Last hour"),
    LAST_24H("Last 24 hours"),
    ALL_TIME("All time");

    /** Epoch-millis cutoff: history at or after this is deleted. ALL_TIME = 0 (everything). */
    fun cutoff(now: Long): Long = when (this) {
        LAST_HOUR -> now - 3_600_000L
        LAST_24H -> now - 86_400_000L
        ALL_TIME -> 0L
    }
}
```

### `data/HistoryDao.kt`
```
@Query("DELETE FROM history WHERE visitedAt >= :cutoff")
suspend fun clearSince(cutoff: Long)
```
(Across every Orbit, matching the existing all-Orbits `clearAll` used by this path.) Fake mirror.

### `BrowserViewModel.kt`
Replace `onClearAllHistory()` at the dialog callsite with:
```
fun onClearHistory(range: ClearDataRange) {
    viewModelScope.launch {
        if (range == ClearDataRange.ALL_TIME) historyDao.clearAll()
        else historyDao.clearSince(range.cutoff(System.currentTimeMillis()))
    }
}
```
(`onClearAllHistory` may be kept if other callers exist, or folded in — check callers.)

### `ui/SettingsScreen.kt` — the clear-data dialog
- Add a `SingleChoiceSegmentedButtonRow` (or radio list) of the three ranges, default `LAST_HOUR`,
  in local `remember` state.
- Dialog copy updated: history is cleared for the chosen range; "Cookies and cached files are always
  cleared." (Honest about the cache limitation.)
- Confirm calls `viewModel.onClearHistory(range)` + the existing `onClearBrowsingData()` (cache) and
  toasts.

## Testing
`ClearDataRangeTest` (pure): `cutoff` for each range (LAST_HOUR = now-3.6e6, LAST_24H = now-8.64e7,
ALL_TIME = 0), monotonic ordering (hour cutoff > 24h cutoff > 0).
`BrowserViewModelTest`: seed history at `now` and `now - 2 days`; `onClearHistory(LAST_24H)` removes
the recent row, keeps the 2-day-old one; `onClearHistory(ALL_TIME)` removes both.

## Non-goals
Per-data-type toggles (cookies/cache/history checkboxes); range-scoped cookie/cache clearing (no
WebView API); per-Orbit range clear (this is the all-Orbits Settings path, matching today).
