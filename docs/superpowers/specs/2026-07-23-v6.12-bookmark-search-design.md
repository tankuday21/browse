# v6.12 — Bookmark search

**Goal:** A search field at the top of the Bookmarks screen that filters the list by title or URL
as you type.

**Why:** Bookmark lists grow; folders (v6.10) help, but scanning is still slow. History already has
search; bookmarks don't. This closes the gap.

## Design

In-memory filter over the already-collected `bookmarks` StateFlow (no DAO round-trip; the list is
Orbit-scoped and already in memory). A pure matcher makes it testable.

### `browser/BookmarkSearch.kt` (pure, JVM-tested)
```
object BookmarkSearch {
    /** Case-insensitive substring match on title OR url; a blank query matches everything. */
    fun matches(bookmark: Bookmark, query: String): Boolean

    /** Filter preserving input order; blank query returns the list unchanged. */
    fun filter(bookmarks: List<Bookmark>, query: String): List<Bookmark>
}
```
Trim the query; blank → no filtering. Match is `title.contains(q, ignoreCase=true) ||
url.contains(q, ignoreCase=true)`.

### `ui/BookmarksScreen.kt`
- Add a search `OrbitTextField` (Search leading icon) below the top bar, `query` in local
  `remember` state. Shown only when there is ≥1 bookmark.
- When `query` is blank: the existing folder-grouped view (`BookmarkFolders.sections`) unchanged.
- When `query` is non-blank: a **flat** filtered list (folders ignored while searching — matches
  are what matter), via `BookmarkSearch.filter(bookmarks, query)`; each row keeps its move/delete
  actions. An empty result shows a small "No bookmarks match \"<query>\"" line.

### Interaction with v6.10 folders
Search is a view mode on top of the same data; it doesn't touch folders. Blank query restores the
grouped view exactly as before.

## Testing (`BookmarkSearchTest`, pure)
- `matches`: title hit, url hit, case-insensitivity, no-match, blank query matches all.
- `filter`: filters by title and url, preserves order, blank returns unchanged, no-match → empty.

## Non-goals
Fuzzy/typo matching; searching folder names; ranking; DAO-level search (in-memory is enough for an
Orbit's bookmark count).
