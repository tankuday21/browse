# v6.13 — Reader font family (serif / sans-serif)

**Goal:** Let the user choose a serif or sans-serif typeface in reader mode, persisted like the
existing reader theme/scale/width settings.

**Why:** Serif vs sans is the most-requested reading preference after size/theme; many readers find
serif more comfortable for long articles. Reader mode already exposes scale, theme, and width — font
family is the missing sibling.

## Design

Mirror the existing reader settings exactly (theme is the closest analog: an enum persisted by name).

### `data/SettingsRepository.kt`
- New `enum class ReaderFont { SANS, SERIF }` (next to `ReaderTheme`).
- `readerFont: Flow<ReaderFont>` + `setReaderFont(font)`, key `stringPreferencesKey("reader_font")`,
  default `SANS` (current behavior). Fake mirror.

### `browser/ReaderMode.kt`
`buildReaderHtml` gains a `font: ReaderFont` param and sets the body `font-family` from it:
- SANS → `-apple-system,Roboto,sans-serif` (today's hardcoded value).
- SERIF → `Georgia,'Times New Roman',serif`.
The `pre,code` block stays monospace regardless.

### `BrowserViewModel.kt`
- `readerFont: StateFlow<ReaderFont>` (Eagerly, SANS) + `onReaderFontSelected(font)`.
- Backup/restore: add `readerFont` to `buildBackupJson` and the restore map (parity with
  `readerTheme`).

### Reader UI (`ui/ReaderOverlay.kt`, `ui/ReadingListScreen.kt`, shared `ReaderControls`)
- Both call sites already collect `readerFontScale`/`readerTheme`/`readerWide` and pass them to
  `buildReaderHtml` and to the shared `ReaderControls`. Add `readerFont` collection at both, pass to
  `buildReaderHtml`, and add `font` + `onFont` to `ReaderControls`.
- `ReaderControls` gains a small segmented/toggle control (Aa-serif vs Aa-sans, or a labeled
  two-option row) wired to `onReaderFontSelected`, styled like the existing theme control.

## Testing
`ReaderModeTest`:
- `buildReaderHtml(font = SERIF)` contains `Georgia` / `serif`; `font = SANS` contains
  `-apple-system` and not `Georgia`.
`ReaderFont` round-trips via settings (fake) — covered by the VM/settings pattern; a focused
settings assertion optional (enum-by-name mirrors readerTheme which is already trusted).

## Non-goals
Arbitrary font-family entry; bundled webfonts (CSP/size); dyslexia-friendly faces; per-article
override.
