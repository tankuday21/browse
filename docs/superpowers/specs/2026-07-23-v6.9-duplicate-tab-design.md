# v6.9 — Duplicate tab

**Goal:** Long-press a tab in the switcher → "Duplicate" → a new tab opens on the same URL,
inheriting the source tab's Orbit, incognito state, and group.

**Why:** A common browser action (Chrome/Firefox both have it) — open a second copy of the current
page to branch off without losing your place. Andromeda has reopen-closed and new-tab, but no
duplicate.

## Design

Pure delegation to the existing `TabManager.newTab`, which already handles id allocation,
persistence, incognito (negative ids, never persisted), and Orbit stamping.

### `BrowserViewModel.kt`
```
/** Duplicate a tab: a new foreground tab on the same URL, same Orbit/incognito/group. */
fun onDuplicateTab(id: Long) {
    val src = tabs.value.find { it.id == id } ?: return
    viewModelScope.launch {
        tabManager.newTab(
            url = src.url,
            incognito = src.isIncognito,
            groupId = src.groupId,
            orbitId = src.orbitId,
        )
    }
}
```
The new tab appends at the end (consistent with every other new-tab path; adjacent-insert is not
how this app orders tabs). The new WebView loads `src.url` through the normal tab-open flow.

### `ui/TabSwitcherScreen.kt`
Add a "Duplicate" `DropdownMenuItem` (ContentCopy icon) to the shared `TabContextMenu`, threaded
as an `onDuplicate: () -> Unit` callback from the grid/list call site
(`{ viewModel.onDuplicateTab(tab.id) }`) through `TabCard`/`TabListRow` exactly like
`onTogglePinned`.

## Testing (`BrowserViewModelOrbitTest` / tab tests)
- Duplicating a normal tab creates a second tab with the same URL and Orbit (count +1).
- Duplicating an incognito tab yields an incognito tab (negative id) with no persisted Orbit.
- Duplicating preserves the source's groupId.

## Non-goals
Adjacent insertion; duplicating scroll position / form state / full history stack (a fresh load of
the URL is the scope, matching "open in new tab").
