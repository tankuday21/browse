# v6.11 — Reader-mode reading-time estimate

**Goal:** Show an estimated reading time ("5 min read") under the title in reader mode.

**Why:** A small, expected touch in a reading view — sets expectations before the user commits.
Reader mode already extracts clean article HTML; the estimate is a pure function of that text.

## Design

Reader HTML is assembled in Kotlin by `ReaderMode.buildReaderHtml(title, contentHtml, …)`. Compute
the estimate from `contentHtml` there and inject a byline line under the `<h1>`.

### `browser/ReadingTime.kt` (pure, JVM-tested)
```
object ReadingTime {
    private const val WORDS_PER_MINUTE = 200

    /** Word count of [html] with tags stripped and whitespace collapsed. */
    fun wordsInHtml(html: String): Int

    /** ceil(words / 200), floored at 1 when there are any words; 0 for none. */
    fun minutes(words: Int): Int

    /** "N min read", or null when there are no words (don't show a bogus 0/1 for an empty body). */
    fun label(html: String): String?
}
```
- `wordsInHtml`: strip `<...>` tags (regex), decode nothing else (word *count* only), split on
  whitespace, drop empties.
- `minutes`: `ceil(words / 200.0)`, min 1 for words > 0 (200 wpm is the standard adult silent
  reading estimate; documented constant).
- `label`: `words == 0 → null` else `"${minutes(words)} min read"`.

### `browser/ReaderMode.kt`
In `buildReaderHtml`, after the `<h1>`, insert (only when `ReadingTime.label(contentHtml)` is
non-null) a muted byline:
```
<div class="reading-time">$label</div>
```
plus a `.reading-time { opacity:.6; font-size:15px; margin:-8px 0 24px; }` style rule that inherits
the reader's theme color. The label text is escaped like the title.

## Testing (`ReadingTimeTest`, pure)
- `wordsInHtml`: strips tags (`<p>one two</p><p>three</p>` → 3), collapses whitespace/newlines,
  ignores attributes (`<a href="x y z">one</a>` → 1), empty/tags-only → 0.
- `minutes`: 0 → 0, 1 → 1, 200 → 1, 201 → 2, 400 → 2, 401 → 3.
- `label`: 0 words → null, 200 words → "1 min read", 450 words → "3 min read".

## Non-goals
Per-language WPM tuning; counting images/code; showing it anywhere but reader mode; a live
scroll-progress indicator.
